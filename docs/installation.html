1System Requirements

Hardware Requirements

2x Microchip LAN9662 boards - VelocityDRIVE evaluation boards
Linux PC - Ubuntu 22.04 LTS or later (kernel 5.15+)
Network Interfaces - Minimum 2 Gigabit Ethernet ports
USB-to-Serial adapter - For console access (115200 baud)
Ethernet cables - CAT5e or better (4 cables minimum)

Software Requirements

Python 3.8+ - For test scripts and automation
Git - For repository management
Bridge utilities - For Linux bridge configuration
TCPdump/Wireshark - For packet analysis

2Hardware Setup

Physical Connections

Connect the boards according to the following topology:

┌──────────┐         ┌─────────────┐         ┌─────────────┐
│    PC    │────────▶│ Sender Board│────────▶│Receiver Board│
│          │  eth3   │  (LAN9662)  │ eth1/2  │  (LAN9662)  │
└──────────┘         └─────────────┘         └─────────────┘
      │                                              │
      └──────────────────────────────────────────────┘
                    Management Network (eth0)
Connection Details:
  • PC enp2s0 ↔ Sender Board eth3 (Access port)
  • Sender Board eth1 ↔ Receiver Board eth1 (FRER path 1)
  • Sender Board eth2 ↔ Receiver Board eth2 (FRER path 2)
  • PC enp11s0 ↔ Receiver Board eth0 (Management)

Serial Console Setup

# Check for serial device
ls -la /dev/ttyUSB*

# Set permissions
sudo chmod 666 /dev/ttyUSB0

# Connect via screen
screen /dev/ttyUSB0 115200

# Or use minicom
minicom -D /dev/ttyUSB0 -b 115200

3Software Installation

Clone Repository

git clone https://github.com/hwkim3330/frer-test.git
cd frer-test

Install Dependencies

# System packages
sudo apt-get update
sudo apt-get install -y \
    python3-pip \
    python3-venv \
    bridge-utils \
    ethtool \
    tcpdump \
    wireshark \
    net-tools \
    iperf3

# Python packages
pip3 install --break-system-packages \
    pyserial \
    pandas \
    numpy \
    matplotlib \
    plotly \
    scapy \
    PyYAML

Verify Installation

# Check Python version
python3 --version

# Verify packages
python3 -c "import serial, pandas, plotly; print('✅ All packages installed')"

# Check network tools
which tcpdump bridge ethtool

4Network Configuration

PC Network Setup

# Configure sender-side interface
sudo ip addr add 10.0.100.2/24 dev enp2s0
sudo ip link set enp2s0 up

# Configure management interface
sudo ip addr add 169.254.13.30/16 dev enp11s0
sudo ip link set enp11s0 up

# Verify configuration
ip addr show enp2s0 enp11s0

Board Initial Setup

# Access board via SSH
ssh root@169.254.100.2

# Or via serial console
screen /dev/ttyUSB0 115200

5Verification

Connectivity Test

# Test management connectivity
ping -c 2 169.254.100.2

# Test data plane connectivity
ping -c 2 10.0.100.1

# Check link status
ethtool enp2s0 | grep "Link detected"
ethtool enp11s0 | grep "Link detected"

Board Status Check

# Via SSH to receiver board
ssh root@169.254.100.2 'ip link show'
ssh root@169.254.100.2 'bridge vlan show'
ssh root@169.254.100.2 'vcap -v is1'

6Quick Start

Ready to test! Follow these steps to run your first FRER test:

Automated Setup

# Configure sender board
python3 setup_sender_serial.py

# Quick connectivity test
./quick_test.sh

# Run complete FRER test
python3 test_frer_complete.py

Monitor Results

# Watch FRER statistics
watch -n 1 'ssh root@169.254.100.2 "frer cs 0 --cnt"'

# Generate visualization
python3 create_visualizations.py

# View results
firefox docs/index.html
