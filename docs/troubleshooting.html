Troubleshooting Guide

Quick Diagnostic Command
bash -c "$(curl -fsSL https://hwkim3330.github.io/frer-test/diagnostic.sh)"
This script will automatically check your setup and identify common issues.

Symptoms:
  • FRER counters show all zeros
  • No packets passing through compound stream
  • VCAP hit counters remain at 0
Diagnostic Steps:
1 Check VCAP Rules
vcap get 1001

Verify Counter and Hit values are incrementing

2 Verify Physical Connections
ip link show | grep "state UP"

All interfaces should show state UP

3 Check VLAN Configuration
bridge vlan show

Ensure VLAN 10 is configured on all ports

Solution:
  1. Recreate VCAP rules with correct port masks
  2. Verify cable connections between boards
  3. Check if traffic is actually being sent to the correct interface
  4. Ensure VLAN tagging matches between sender and receiver
# Reset and reconfigure vcap del 1001 vcap add 1001 is1 10 1 VCAP_KFS_NORMAL \ IF_IGR_PORT_MASK 0x008 0x1ff \ ETYPE 0x0800 = \ VCAP_AFS_S1 ISDX_REPLACE_ENA 1 ISDX_ADD_VAL 1

Symptoms:
  • ssh: connect to host 169.254.100.2 port 22: Connection refused
  • No route to host
  • Timeout when trying to connect
Diagnostic Steps:
1 Check ARP Table
arp -n | grep 169.254.100
2 Verify IP Configuration
ip addr show enp11s0
Solution:
# Configure management interface sudo ip addr add 169.254.13.30/16 dev enp11s0 sudo ip link set enp11s0 up # Test connectivity ping -c 2 169.254.100.2 # If still failing, use serial console screen /dev/ttyUSB0 115200

Symptoms:
  • No frames with EtherType 0xF1C1 captured
  • FRER generation counter not incrementing
  • Duplicate frames not created
Diagnostic:
# Check for R-TAG frames tcpdump -i eth1 -n -e ether proto 0xf1c1 -c 10 # Verify FRER generation config frer iflow 1
Solution:
# Ensure generation is enabled frer iflow 1 --generation 1 --dev1 eth1 --dev2 eth2 # Verify ISDX mapping vcap get 1001 # Should show ISDX_ADD_VAL: 1

Symptoms:
  • CPU usage above 50%
  • Slow SSH response
  • Packet drops under load
Performance Tuning:
# Check system load top -b -n 1 | head -20 # Disable unnecessary flooding bridge link set dev eth1 flood off mcast_flood off bridge link set dev eth2 flood off mcast_flood off # Optimize FRER parameters frer cs 0 --hlen 10 --reset_time 500

Symptoms:
  • Same packet received multiple times at destination
  • DiscardedPackets counter remains at 0
  • Elimination rate below 90%
Root Causes:
Component Check Expected
Compound Stream frer cs 0 enable: 1
Member Streams frer ms eth1 ID enable: 1, cs_id: 0
Ingress Flows frer iflow 3 ms_enable: 1, pop: 1
Complete Fix:
# Reset and reconfigure elimination frer cs 0 --enable 1 --alg 0 --hlen 10 --reset_time 500 frer ms eth1 28 --enable 1 --alg 1 --reset_time 500 --cs_id 0 frer ms eth2 30 --enable 1 --alg 1 --reset_time 500 --cs_id 0 frer iflow 3 --ms_enable 1 --ms_id 28 --pop 1 --dev1 eth3 frer iflow 4 --ms_enable 1 --ms_id 30 --pop 1 --dev1 eth3
